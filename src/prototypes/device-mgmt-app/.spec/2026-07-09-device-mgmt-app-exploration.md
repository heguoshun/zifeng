# 设备管理 App 需求探索快照

> 探索日期：2026-07-09  
> 目标原型：`device-mgmt-app`（新建移动端原型，配套现有 `iot-platform-home` PC 平台）

## 已确认决策

### 产品定位

- **关系**：现场运维移动端，作为「紫峰装备智慧化管理平台」PC 端的配套 App。
- **目标用户**：现场运维人员（泵房、管网巡检、设备维护等户外/现场场景）。
- **核心任务**：查看水表、压力计等设备信息与实时数据；处理告警；跟进并完成工单。
- **设计方向**：延续现有 PC 平台品牌色与管理语义，适配移动端底部 Tab 布局。

### 使用场景

- 设备、告警、工单**三者同等重要**，首页与各 Tab 均需突出。
- 主路径覆盖：告警/工单驱动处置 + 主动查询设备，不做单一主场景偏废。

### 功能范围（首版）

| 模块 | 纳入 | 说明 |
|------|------|------|
| 设备 | ✅ | 列表、搜索/筛选、详情、实时读数；水表（读数、在线、地址、用户号/表身号）、压力计（压力值、在线、位置、阈值相关状态） |
| 告警 | ✅ | 列表（待处理/处理中/已处理）、详情、确认已知晓、关闭（误报/已恢复）、转工单 |
| 工单 | ✅ | 我的待办、列表（待接单/处理中/已完成）、接单、填写处理结果、完结 |
| 登录与租户 | ✅ | 沿用 PC 账号体系；数据按租户隔离 |
| 消息提醒 | ✅ | App 内待办角标 + 告警/工单通知列表（原型可用模拟数据） |

### 导航结构

底部 **5 Tab**：

1. **首页** — 待办摘要（待处理告警、待办工单、异常设备）+ 快捷入口
2. **设备** — 设备列表与详情
3. **告警** — 告警列表与处置
4. **工单** — 工单列表与处置
5. **我的** — 账号、消息通知、设置

### 非目标（首版不做）

- 设备建档、产品配置、协议管理
- 告警规则配置、告警等级管理
- 用户/角色/租户管理
- 用水分析、设备综合视图等管理报表
- 地图定位、扫码查设备
- 现场拍照/附件上传、派单/转派
- 新建工单（首版由告警转工单或 PC 端创建）

### 与现有 PC 平台的关系

- 数据模型与业务语义对齐 `iot-platform-home`（设备、告警、工单数据结构可复用/参考）。
- App 侧重「看 + 核心处置」；复杂配置与管理报表留在 PC 端。

## 待确认 / 开放问题

- 原型正式名称与对外标题（工作 ID：`device-mgmt-app`）。
- 工单「填写处理结果」的字段口径（纯文字 vs 结构化选项）。
- 告警「转工单」是自动创建还是关联已有工单。

## 建议验收口径

1. 登录后可进入 5 Tab 导航，首页展示三类待办摘要数字。
2. 设备 Tab 可按类型/区域筛选，水表与压力计详情展示对应关键读数。
3. 告警 Tab 可筛选状态，待处理告警可执行确认、关闭、转工单。
4. 工单 Tab 可看待办，执行接单 → 填处理说明 → 完结闭环。
5. 我的 Tab 有消息通知列表，新告警/工单有角标或列表提醒。
6. 视觉风格与 PC 平台品牌一致，状态色/告警等级色语义统一。
