# 远传升级功能审核机制优化说明

## 📋 优化概述

为远传升级功能添加了完整的审核机制，参考告警工单业务闭环逻辑，实现任务创建 → 待审核 → 审核通过/驳回 → 执行的状态流转。

## ✨ 主要改动

### 1. 数据类型扩展 (`data/remoteUpgrade.ts`)

**新增字段：**
- `UpgradeTaskStatus`: 新增 `'待审核' | '审核驳回'` 状态
- `auditor`: 审核人字段
- `auditTime`: 审核时间字段
- `auditRemark`: 审核备注字段

**新增常量：**
- `UPGRADE_TASK_STATUS_OPTIONS`: 任务状态选项列表

### 2. 任务创建逻辑修改 (`pages/RemoteUpgradePage.tsx`)

**改动点：**
- 新建任务默认状态从 `'执行中'/'待执行'` 改为 `'待审核'`
- 提示信息更新为 "升级任务创建成功，等待审核"
- 新增 `handleResubmitTask` 函数，支持重新编辑被驳回的任务

### 3. 审核对话框组件 (`components/AuditDialog.tsx`)

**新功能：**
- 独立的审核对话框组件
- 支持审核通过和驳回操作
- 可填写审核备注（选填）
- 显示任务名称和审核说明

### 4. 任务详情页增强 (`components/UpgradeTaskDetailView.tsx`)

**新增功能：**

#### a) 待审核任务列表
- 在固件包信息卡片下方显示待审核任务
- 显示任务基本信息和状态标签
- 提供审核操作入口
- 显示审核备注信息

#### b) 审核提示卡片
- 有待审核任务时显示黄色警告卡片
- 提示有待审核任务数量
- 引导用户进行审核操作

#### c) 批次列表审核按钮
- 在批次操作列动态显示"审核"按钮
- 仅对待审核的批次显示审核按钮
- 已取消或已完成的任务不显示审核按钮

#### d) 状态标签组件
- 新增 `TaskStatusTag` 组件
- 不同状态使用不同颜色标识：
  - 待审核: 橙色 (#e6a23c)
  - 审核驳回: 红色 (#f56c6c)
  - 待执行: 灰色 (#909399)
  - 执行中: 蓝色 (#409eff)
  - 已完成: 绿色 (#67c23a)
  - 部分失败: 红色 (#f56c6c)

### 5. 主列表页优化 (`pages/RemoteUpgradePage.tsx`)

**新增功能：**
- 统计每个固件包的待审核任务数量
- 在"任务详情"按钮上显示红色角标，提示待审核任务数
- 传递审核相关回调函数给详情页

## 🔄 状态流转逻辑

```
创建任务 
    ↓
待审核 (初始状态)
    ↓
    ├─→ 审核通过 → 根据升级时间设置状态
    │              ├─ 立即升级 → 执行中
    │              └─ 定时升级 → 待执行
    │
    └─→ 审核驳回 → 可重新编辑提交
                   ↓
               待审核 (重新进入审核流程)
```

## 🎯 用户体验改进

1. **明确的审核提示**
   - 任务创建后明确提示"等待审核"
   - 固件包列表显示待审核任务数量角标
   - 详情页顶部显示醒目的审核提示卡片

2. **便捷的审核操作**
   - 待审核任务集中展示
   - 一键审核，支持填写备注
   - 审核结果即时反馈

3. **灵活的驳回处理**
   - 被驳回的任务可以重新编辑
   - 保留原有配置，只需修改必要内容
   - 重新提交后进入新一轮审核

4. **清晰的状态标识**
   - 彩色状态标签直观显示任务状态
   - 审核备注集中展示，便于追溯
   - 审核人和审核时间记录完整

## 📝 使用流程

### 普通用户（创建任务）
1. 选择固件包，点击"创建任务"
2. 配置升级参数（版本、范围、时间等）
3. 提交任务，系统提示"等待审核"
4. 可在任务详情页查看审核状态
5. 如被驳回，点击"重新编辑"修改后再次提交

### 管理员（审核任务）
1. 进入有 pending 任务的固件包详情页
2. 查看"待审核任务"列表
3. 点击"审核"按钮
4. 查看任务详情，填写审核备注（可选）
5. 点击"通过"或"驳回"
6. 审核通过后任务自动进入执行队列

## 🔧 技术实现要点

1. **状态管理**: 通过 React state 管理任务和审核状态
2. **数据关联**: 通过 `packageId` 关联固件包和任务
3. **条件渲染**: 根据任务状态动态显示操作按钮
4. **表单验证**: 审核对话框支持可选备注输入
5. **Toast 提示**: 所有关键操作都有即时反馈

## ⚠️ 注意事项

1. 新创建的任务不会立即执行，必须经过审核
2. 审核通过后的任务会根据设置的升级时间自动执行
3. 被驳回的任务需要删除旧任务后重新创建
4. 审核备注会永久保存在任务记录中，便于追溯
5. 模拟环境中审核人固定为"管理员"，实际部署时需接入真实用户系统
