## 1. 产品概述

### 1.1 产品定位

紫峰装备智慧化管理平台是一套面向物联网设备全生命周期管理的 B 端后台系统，覆盖设备接入、大表用水监测、告警处置、消息推送与平台运维配置。平台以「资产可见、运行可控、异常可追、处置可闭环」为管理主线，帮助设备运维管理层在普通办公电脑上完成日常管理工作。

### 1.2 产品 slogan

智慧物联 · 高效运维 · 安全可靠

### 1.3 核心价值

- **统一接入**：产品建模、设备注册、协议与证书配置在同一平台完成。
- **运行可视**：综合视图、地图分布、数据监测与报表支撑日常巡检。
- **风险闭环**：告警规则、等级管理、工单流转形成处置链路。
- **消息触达**：订阅、模板与推送源配置保障关键事件通知。
- **多租户治理**：租户、组织、权限与系统参数支撑平台规模化运营。

### 1.4 标准业务主线

设备接入遵循「**创建产品 → 定义物模型 → 配置接入 → 添加设备 → 设备上线**」链路；大表业务遵循「**区域配置 → 设备绑定 → 数据监测 → 异常告警 → 工单处置 → 档案留痕**」链路。

## 4.1 登录与认证

### 业务目标

保障平台入口安全，建立用户会话与权限上下文，为后续模块的数据域与功能权限控制提供基础。

### 业务对象

| 对象 | 关键字段 | 说明 |
| --- | --- | --- |
| 登录凭证 | account, password, captcha | 账号密码 + 图形验证码 |
| 会话用户 | account, displayName, tenantId, roleId, permissionIds, alarmRuleCategoryScopes | 登录成功后写入会话，驱动菜单与操作权限 |

### 主流程

1. 用户打开平台 → 未登录则展示登录页。
2. 输入账号、密码，填写图形验证码（4 位字母数字，不区分大小写）。
3. 系统校验：账号非空、密码非空、验证码正确、账号存在、密码匹配、账号未冻结。
4. 校验通过 → 写入会话（勾选「记住账号」则持久化到 localStorage，否则 sessionStorage）→ 跳转「设备综合视图」。
5. 用户点击退出 → 清除会话 → 返回登录页。

### 业务规则

| 规则编号 | 规则描述 |
| --- | --- |
| AUTH-01 | 账号状态为「冻结」时拒绝登录，提示账号已冻结 |
| AUTH-02 | 密码错误或无此账号时统一提示「账号或密码错误」，不暴露具体原因 |
| AUTH-03 | 新用户默认初始密码为系统参数配置值（原型默认 `12345678`） |
| AUTH-04 | 超级管理员账号不可被冻结、不可被删除 |
| AUTH-05 | 登录页提供「联系管理员」入口，展示管理员姓名、电话、邮箱、工作时间 |

### 异常与边界

- 会话过期或本地会话被清除 → 强制回登录页。
- 生产环境需补齐：会话续期、多端登录策略、密码强度与重置流程。

## 4.2 设备接入

设备接入模块负责 IoT 资产从建模到上线运维的全链路管理。模块内存在三条并行概念线：**产品/物模型线**、**接入配置线（协议/证书/网络服务）**、**设备实例运营线**。

### 4.2.1 设备综合视图

#### 业务目标

让设备运维管理层在 10 秒内判断：当前有多少资产、运行是否健康、数据质量是否有风险、告警工单是否闭环。

#### 信息主线

```text
资产家底 → 运行健康 → 数据质量 → 告警与工单闭环
```

#### 页面结构

面向 1440px 左右办公屏幕，采用纵向工作台布局，允许滚动；核心结论首屏可见。

| 区块 | 内容 |
| --- | --- |
| 页头 | 标题、数据更新时间、手动刷新按钮、运行状态标识 |
| 全局筛选 | 统计时间（今日 / 近 7 天 / 近 30 天）、产品树筛选、指标口径说明 |
| 设备运行概况 | 设备资产块 + 在线率 / 数据正常率 / 告警处理率 |
| 第二段 | 设备资产概况 + 离线分析（双列） |
| 第三段 | 数据接收异常 + 告警与工单（双列） |
| 底部 | 设备接入流程快捷入口（5 步） |

#### 业务对象与指标口径

| 指标 | 业务含义 | 计算/展示口径 | 健康阈值 |
| --- | --- | --- | --- |
| 设备总量 | 已建档的全部设备 | 当前快照 | — |
| 在用设备 | 已投入业务使用的资产 | enabled 且非 disabled | 在用率 = 在用 / 总量 |
| 闲置设备 | 已建档但未投入使用 | 未启用或 status=disabled | — |
| 在线率 | 当前连接健康程度 | 在线设备数 / 设备总量 | 正常 ≥98%，关注 95~98%，异常 <95% |
| 数据正常率 | 按要求正常上报的设备占比 | 异常设备去重计数 | 正常 ≥99%，关注 97~99%，异常 <97% |
| 告警处理率 | 已处理告警占告警总量比例 | 已处理 / 告警总量 | 正常 ≥95%，关注 85~95%，异常 <85% |

**口径隔离要求**：

- 「在用/闲置」描述资产业务使用状态；「在线/离线」描述连接状态，两者不可混用。
- **快照类指标**（设备总量、当前在线/离线台数、当前异常设备、当前待处理告警）不随统计时间切换变化。
- **趋势类指标**（在线率、数据正常率、告警处理率及环比）随统计时间筛选变化；数据接收异常趋势图按所选周期展示历史波动。

#### 主流程

1. 用户进入综合视图，默认展示「近 7 天」统计范围、全部产品。
2. 可选全局筛选：**统计时间**、**产品**（产品分类树，含全部）。
3. 首屏「设备运行概况」：左侧设备资产（总量及在用/闲置拆分），右侧三项健康指标（核心数值、同期变化、状态标签及辅助说明）。
4. 点击概况中的在用/闲置、在线/离线、异常设备、待处理告警 → 打开**明细抽屉**（设备列表 + 分页 + CSV 导出）。
5. 向下滚动：资产概况（环形占比 + 分类明细，可切换按产品/按分组）→ 离线分析（离线设备按产品分布）→ 数据接收异常（当前快照 + 异常分类 + 趋势条）→ 告警与工单（告警/工单状态环形占比，可跳转对应模块）。
6. 底部「设备接入流程」：创建产品 → 定义物模型 → 配置接入（协议/证书/网络服务）→ 添加设备 → 设备上线。

#### 体验与交互要求

| 要求 | 说明 |
| --- | --- |
| 指标口径 | 筛选栏提供「指标口径」浮层，说明各指标计算方式及快照/趋势区分 |
| 状态标签 | 统一为：正常、关注、异常、无数据；悬浮展示阈值图与当前值位置，不可仅依赖颜色 |
| 资产维度切换 | 未选单一产品时，可在「按产品 / 按分组」间切换；选中单一产品时自动切为「按分组」 |
| 环形占比图 | 资产概况、离线分析、告警与工单均使用交互式环形图，悬浮展示分类名称、数量与占比 |
| 明细抽屉 | 展示口径总数、列表数据量、统计范围；支持分页与 CSV 导出 |
| 不可下钻项 | 数据异常分类（解析失败、缺报、非整点上报）暂不提供跨页跳转 |
| 刷新 | 手动刷新更新「数据更新时间」，刷新过程展示加载态 |

#### 业务规则

| 规则编号 | 规则描述 |
| --- | --- |
| OV-01 | 全局筛选对页面全部模块生效；下钻明细保留当前筛选上下文 |
| OV-02 | 趋势类指标跟随时间筛选；资产存量类指标显示当前快照 |
| OV-03 | 对比周期：今日比昨日，7 天比前 7 天，30 天比前 30 天 |
| OV-04 | 数据异常类指标暂不提供跨页下钻，避免无明细页时的误导跳转 |
| OV-05 | 状态标签统一为：正常、关注、异常、无数据；不可仅依赖颜色表达 |
| OV-06 | 概况区健康指标（在线率/数据正常率/告警处理率）不使用进度条，不在指标卡片上配置跳转箭头 |
| OV-07 | 告警环形图点击跳转告警列表；工单环形图点击跳转工单管理 |

#### 数据接收异常分类

| 异常类型 | 业务含义 |
| --- | --- |
| 解析失败 | 设备上报数据无法按物模型解析 |
| 缺报 | 在应上报时段内未收到数据 |
| 非整点上报 | 数据到达时间偏离整点容差（容差待后端确认） |

同一设备在统计周期内多次异常：**异常设备数去重**，**异常次数累计**，两者不可混用。

### 4.2.2 产品管理

#### 业务目标

定义设备「是什么」——通过产品与物模型描述设备类型、接入协议、可读属性、可调用功能与可上报事件，为设备实例化提供模板。

#### 产品分类体系

产品分类为独立维护的多级树，与产品列表左侧筛选联动，并作为角色**数据域**、告警规则分类、设备筛选等产品维度的统一口径。分类本身需要明确归属的**业务类型**，用于判断该分类下产品进入哪个业务域。

```text
全部
 └─ 水务设备
     ├─ 大表
     │   ├─ 电磁表 / 机械表 / 超声表 / 流量计
     ├─ 户表
     ├─ 压力计
     ├─ 水质仪
     └─ 智慧水站
```

- 产品管理页左侧展示分类树及各分类下产品数量。
- 「编辑分类」跳转**产品分类**子页，支持分类 CRUD（分类编号由系统生成；名称、上级分类、业务类型、备注）；有子分类时不可删除。
- 新增/编辑分类以右侧抽屉形式打开，抽屉内可选择该产品分类所属业务类型。
- 业务类型来自系统管理「字典管理」中的**业务类型**字典，默认包含大表、户表、压力计、水质仪、智慧水站等。
- 大表类产品可按子类型（电磁表/机械表/超声表/流量计等）及供应商（山科、惠然、汇中等）进一步区分。
- 判断某设备是否属于大表业务，不依赖分类 ID 是否为 `dabiao`，也不依赖分类名称是否叫“大表”；只依据该设备所属产品的产品分类归属业务类型是否为「大表」。

#### 业务对象

**产品（Product）**

| 字段 | 必填 | 说明 |
| --- | --- | --- |
| code | 系统生成 | 产品编号，新建时随机 `P######` |
| name | 是 | 产品名称 |
| categoryId | 是 | 产品分类（树形选择，对应上述分类体系） |
| nodeType | 是 | 直连设备 / 网关设备 / 网关子设备 |
| vendor | 否 | 供应商，默认「紫峰装备」；大表类可选预设厂家列表 |
| protocolId | 是 | 绑定的网络协议组件 |
| deviceCount | 系统维护 | 关联设备数，用于删除保护 |
| properties / functions / events | 否 | 物模型三要素 |
| remark | 否 | 备注，列表中截断展示 |

**产品分类（ProductCategory）**

| 字段 | 必填 | 说明 |
| --- | --- | --- |
| id/code | 系统生成 | 分类唯一标识/编号，不作为大表业务判断依据 |
| name | 是 | 分类展示名称，可改名，不影响业务归属 |
| parentId | 否 | 上级分类；子分类默认继承上级业务类型，也可按页面能力选择明确业务类型 |
| businessType | 是 | 业务类型，来源于系统字典；大表中心仅处理业务类型为「大表」的产品设备 |
| remark | 否 | 备注 |

**物模型要素**

| 类型 | 关键字段 | 用途 |
| --- | --- | --- |
| 属性 Property | identifier, name, dataType, accessMode（只读/读写） | 设备状态与采集数据 |
| 功能 Function | identifier, name, async, inputJson | 远程控制与下行指令 |
| 事件 Event | identifier, name, eventType（信息/告警/故障） | 设备主动上报 |

#### 主流程

**产品列表**

1. 左侧产品分类树筛选 + 节点类型 / 名称编号筛选 → 查询。
2. 支持**卡片视图**（默认）与**列表视图**切换；分页展示。
3. 工具栏：新增、批量导出、批量导入（原型占位）。
4. 单条操作：查看 / 编辑 / **设备**（跳转设备管理并带 productId 筛选）/ 删除。

**产品分类**

1. 树形列表展示全部分类节点及编号、业务类型、更新时间。
2. 新增/编辑分类：名称、上级分类、业务类型、备注；系统生成分类编号，表单从右侧抽屉打开。
3. 删除前检查是否有子分类；有子分类则禁止删除。

**新增/编辑/查看产品**

1. 独立表单页，支持 create / edit / view 三种模式；view 模式只读。
2. Tab「基本信息」：名称、节点类型、分类（树形选择）、供应商、网络协议、备注。
3. 选择网络协议 → 打开协议选择抽屉（按组件名/服务名/协议名搜索）。
4. Tab「物模型」：维护属性 / 功能 / 事件（抽屉表单 CRUD）；可从物模型库引用标准模型。
5. 保存 → 校验 → 写回产品列表。

#### 业务规则

| 规则编号 | 规则描述 |
| --- | --- |
| PRD-01 | 产品名称、节点类型、产品分类、网络协议为必填 |
| PRD-02 | 删除产品时，若 `deviceCount > 0` 则禁止删除，提示「当前产品下存在关联设备，无法删除！」 |
| PRD-03 | 网关子设备类型产品不参与顶层设备实例分配，仅作为子设备挂载模板 |
| PRD-04 | 新建产品 `deviceCount` 初始为 0 |
| PRD-05 | 物模型属性数据类型支持 int/float/double/text/bool/enum/struct |
| PRD-06 | 产品分类名称在同层级下不可重复 |
| PRD-07 | 产品列表「设备」操作通过 URL 参数 `productId` 深链至设备管理 |
| PRD-08 | 产品分类必须归属一个业务类型；大表业务设备按业务类型识别，不按分类名称或分类 ID 识别 |
| PRD-09 | 系统管理「字典管理」中的原「产品类型」统一调整为「业务类型」，产品分类创建/编辑时引用该字典 |

#### 跨模块关系

- 产品 → 设备管理：决定设备 nodeType、物模型、Tab 结构、远程控制能力。
- 产品 → 远程升级：固件包按 productId 关联。
- 产品 → 大表中心：产品分类归属业务类型为「大表」的产品实例参与大表监测、区域配置与大表档案。
- 产品 → 告警规则 / 设备告警：规则与告警按产品分类树组织；角色数据域与分类树节点 ID 对齐。
- 产品分类 → 系统管理：角色数据域配置引用同一分类树。

### 4.2.3 设备管理

#### 业务目标

管理设备实例全生命周期：注册、定位、分组、启停、远程控制，并与告警、档案、地图、升级联动。

#### 业务对象

**设备（Device）**

| 字段 | 说明 |
| --- | --- |
| code | 设备编号，新建随机 `S######` |
| productId | 所属产品，决定 nodeType 与物模型 |
| gatewayId | 网关子设备指向父网关 |
| status | online / offline / fault / disabled |
| enabled | 启用开关，与 status 联动 |
| groups[], tags[] | 分组标签，格式 `类型:大表`、`区域:城东`、`管网:主管网` |
| longitude, latitude, mapAddress | 地图定位 |
| collectFrequency + unit | 采集频率（秒/分钟/小时） |
| userNo, userName, installAddress 等 | 安装与用户档案 |
| largeMeterAreaId | 大表片区绑定（与大表中心联动） |

#### 主流程

**设备列表**

1. 左侧产品树 + 分组筛选（支持 URL 带 productId / groupId 深链）。
2. 筛选：节点类型、设备状态（在线/离线/故障/禁用）、部门、关键词。
3. 顶部状态统计条展示各状态数量。
4. 卡片/列表操作：启用开关、编辑、远程控制、数据、删除、新增。

**新增/编辑设备**

1. **新建模式**仅「基本信息」Tab；编辑/查看模式按 nodeType 展开全部 Tab。
2. 基本信息：名称、产品、采集频率、定位方式、注册码、启用、分组、地图选点。
3. **网关设备**额外 Tab：子设备管理、设备定时、日志、告警、文件等。
4. **非网关设备**额外 Tab：属性数据、设备影子（网关无影子 Tab）。
5. 保存 → 校验 → 写回设备列表；关键字段变更自动写入大表档案。

**远程控制**

1. 列表或详情点击「远程控制」→ 打开控制抽屉。
2. 系统校验设备已启用 → 展示产品物模型中的可写属性与可调用功能。
3. 下发控制指令（原型为模拟反馈）。

#### 状态流转

```text
启用开关 ON  → status: online，重置在线时长计时
启用开关 OFF → status: disabled，冻结当前在线时长
编辑保存且 enabled=true  → 保留 fault 状态，否则置 online
编辑保存且 enabled=false → 置 disabled
```

**概念隔离**：`enabled` 是业务启停；`online/offline/fault` 是连接与运行态，表单中不可直接编辑连接态。

#### 业务规则

| 规则编号 | 规则描述 |
| --- | --- |
| DEV-01 | 设备名称、所属产品、定位方式为必填 |
| DEV-02 | 手动定位必须在地图选点（经纬度非空）；自动定位清空经纬度 |
| DEV-03 | 已禁用设备不可远程控制，提示「设备已禁用，无法远程控制」 |
| DEV-04 | 网关子设备仅允许挂载到 nodeType=网关设备 的父设备，且不可重复绑定 |
| DEV-05 | 子设备继承父网关的 departmentId / spaceId |
| DEV-06 | 用户/点位/启停状态变更时，若为大表设备，自动生成档案记录 |

#### 跨模块关系

- 设备 ↔ 设备分组：通过 tags 与 groups 关联。
- 设备 → 设备地图：共享坐标，地图可回写位置。
- 设备 → 远程升级：按 productId 筛选升级目标。
- 设备 → 告警/工单：详情 Tab 展示关联告警，可转工单。
- 设备 → 大表档案 / 区域配置：largeMeterAreaId 与安装信息联动。

### 4.2.6 远程升级

#### 业务目标

管理固件版本与 OTA 升级任务，按产品维度向目标设备下发升级；升级任务须经审批后方可进入执行队列。

#### 业务对象

| 对象 | 关键字段 |
| --- | --- |
| 固件包 | name, type（完整包/差分包）, productId, version, baseVersion（差分包必填）, fileName |
| 升级任务 | packageId, scope（全部/指定设备）, deviceIds, scheduledAt, retryStrategy, timeout, status, auditor, auditTime, auditRemark |
| 升级批次 | 批次号, versionBefore/After, deviceCount, status（未开始/进行中/已完成/已取消） |
| 设备升级明细 | 单设备：待升级 → 升级中 → 成功/失败/取消 |

#### 主流程

1. **固件包管理**：上传固件（.zip/.rar/.bin/.apk，≤100MB）→ 关联产品 → 填写版本。
2. **创建升级任务**：选择目标固件 → 选择范围（全部设备或指定设备，指定时至少 1 台）→ 选择升级时间 → 配置重试策略与超时 → 提交后进入待审核。
3. **审批任务**：具备审批权限的用户在「审批任务」工作台通过或驳回；驳回原因必填。
4. **任务执行**：审核通过后生成任务批次并进入待执行；到达升级时间后自动开始执行。
5. **任务详情**：默认查看任务批次；可切换「审批记录」查看审批留痕；可查看单设备升级状态。

#### 页面组织

- 工作台一级 Tab：**固件包管理**（默认）、**审批任务**（唯一审批操作入口）。
- 任务详情二级 Tab：**任务批次**（默认）、**审批记录**。
- 进入任务详情后隐藏一级 Tab；审批记录 Tab 不提供审批按钮。

#### 业务规则

| 规则编号 | 规则描述 |
| --- | --- |
| OTA-01 | 固件包必填：名称、产品、版本、文件；差分包额外必填 baseVersion |
| OTA-02 | 指定设备升级时，设备列表按固件包 productId 过滤 |
| OTA-03 | 升级时间 scheduledAt 必填；不再提供「立即升级 / 定时升级」选项 |
| OTA-04 | 删除固件包级联删除关联升级任务与批次 |
| OTA-05 | 任务状态：待审核 → 待执行 → 执行中 → 已完成 / 部分失败；审核驳回可重新编辑后再次提交 |
| OTA-06 | 待审核、审核驳回任务不生成任务批次；审核通过后才生成批次 |
| OTA-07 | 审核通过后、未到升级时间的批次状态为未开始；到达升级时间后变为进行中 |
| OTA-08 | 任务批次按审核通过时间先后排序 |
| OTA-09 | 未开始批次下全部设备展示为待升级，更新时间为「—」 |
| OTA-10 | 审批操作仅在「审批任务」Tab 提供；首位提交审批结果的用户记为实际审核人 |

### 4.2.7 接入配置（协议管理 / 网络协议 / 证书管理 / 网络服务）

#### 概念区分（PRD 必须明确）

| 模块 | 业务定位 | 创建方式 |
| --- | --- | --- |
| 协议管理 | 底层通信协议定义（Local 本机 / Jar 包） | 表单抽屉 |
| 网络协议 | 服务类型 + 网络组件 + 通信协议的组合实例 | 四步向导 |
| 证书管理 | 设备接入 TLS/DTLS 证书与私钥 | 表单抽屉 |
| 网络服务 | 平台侧网络接入服务（监听地址、端口、粘拆包、MQTT 参数等） | 表单抽屉 |

标准接入配置顺序：**证书（如需 DTLS）→ 网络服务 → 网络协议 → 产品绑定协议**。

#### 协议管理 — 主流程与规则

1. 列表筛选（名称、类型 Local/Jar）→ 新增/编辑/删除。
2. 校验：名称必填且不可重复；Jar 类型必须上传协议文件；Local 类型必须填本机地址。
3. 删除时：若已有产品引用（protocolLabel 或 protocolId 匹配）则禁止删除。

#### 网络协议 — 向导流程

1. 选择服务类型（UDP/TCP/HTTP/CoAP/MQTT/WebSocket/自定义）。
2. 选择网络组件（如水表网络、电表网络等）。
3. 选择通信协议（随步骤 1 过滤可选协议，含 Modbus TCP、DTLS 等）。
4. 填写组件名称（必填、不可重复）、描述（≤100 字）→ 完成。

#### 证书管理 — 规则

- 必填：标准（X.509/国密）、名称、证书文件、私钥；名称不可重复。
- 文件格式：.pem/.crt/.cer/.pfx/.p12，≤100MB。

#### 网络服务 — 条件校验

| 条件 | 必填项 |
| --- | --- |
| 通用 | 服务名（唯一）、IP 地址、端口 |
| 自定义服务 | 必须上传 SDK 组件文件 |
| 开启 DTLS | 必须选择有效证书 |
| TCP 协议 | 必须配置粘拆包规则（不处理/分隔符/固定长度/长度字段） |
| MQTT 协议 | 集群地址、用户名、密码、消息质量必填 |

## 4.3 大表中心

大表中心面向用水监测业务。存在两套关联数据：**监测表（largeMeters）** 用于数据监测/报表/分析展示；**设备实例（devices + largeMeterAreaId）** 用于区域绑定与档案。区域配置操作会按 userNo 同步两者。大表中心只处理产品分类所属业务类型为「大表」的设备，不按产品分类名称或分类 ID 判断。

### 4.3.1 数据监测

#### 业务目标

运维人员日常巡检大表运行状态，快速发现离线、告警、用水异常设备，并进入单表分析。

#### 业务对象 — 监测表

核心字段：身份（name/code/userNo/userName/bodyNo）、areaId、状态、读数（currentReading/dailyUsage/monthlyUsage/nightlyUsage 等）、dataTime、manufacturer/remoteManufacturer。

#### 状态展示规则（重要）

| 底层状态 | 列表展示 |
| --- | --- |
| 在线 | 「在线」 |
| 离线 | 「离线」 |
| 告警 | **展示具体告警类型**（如大流告警、持续用水），不用「告警」统称 |

告警类型枚举：离线、反向流量、低电压告警、大流告警、持续用水、识别不一致、连续三天未上报。

#### 主流程

1. 左侧片区树筛选（含「全部片区」「未分配区域」及子片区聚合）。
2. 筛选：告警类型、表具/远传厂家、关键词（用户号/名称/设备编号/表身号）。
3. 列表或图片（码表卡片）视图浏览，分页。
4. 配置「夜间用水」全局规则：时段（默认 22:00-06:00）、监测字段（夜间用水/占日比例/峰值/谷值）。
5. 点击「数据分析」→ 进入该表的用水分析页。

#### 业务规则

| 规则编号 | 规则描述 |
| --- | --- |
| LM-01 | 夜间用水配置至少选择一个监测字段；起止时间不可相同 |
| LM-02 | 离线设备 dailyUsage 为 0，数据时间偏旧 |
| LM-03 | 列表操作仅保留「数据分析」，合并原「设备详情」「指标分析」 |

### 4.3.2 设备数据（用水分析）

#### 主流程

1. 顶部展示设备身份、所属片区、用户、状态、最后数据时间。
2. 默认展示近一周逐日用水趋势；日用水明细展示日期、抄表时间和日用水量，不展示数据状态。
3. Tab 切换：历史数据 / 日用水 / 月用水 / 时用水 / 夜间用水 / 报警数据。
4. 日用水、月用水、时用水使用趋势图与数据明细视图，支持时间范围切换与 CSV 导出。
5. 夜间用水使用周期统计视图，默认统计本月；支持昨日、本月、本年及自定义日期时间范围切换。
6. 夜间用水顶部独立展示累计夜间用水、峰值流量、谷值流量三张统计卡；峰值、谷值同时展示实际发生日期和时刻。
7. 夜间用水下方展示峰谷时刻分布，横坐标仅覆盖全局夜间时段，并按跨零点顺序排列；例如 22:00-06:00 依次展示 22:00、23:00、00:00 至 06:00。
8. 夜间用水不展示逐日数据明细，不提供趋势图/数据明细切换；统计摘要和分布图均随查询周期刷新。
9. 报警 Tab：展示该设备关联告警，支持直接处理、转工单。
10. 返回 → 回到数据监测列表。

#### 业务规则

| 规则编号 | 规则描述 |
| --- | --- |
| WA-01 | 夜间用水异常判定：夜间用水 > max(日用水×35%, 12) 标记为异常 |
| WA-02 | 无有效 meter 时自动重定向回数据监测 |
| WA-03 | 报警 Tab 按 deviceCode 关联告警实例 |
| WA-04 | 日用水、月用水、时用水均不单独展示数据状态；日用水 CSV 导出仅包含日期、抄表时间和日用水量 |
| WA-05 | 夜间用水默认周期为本月；重置查询条件后恢复本月 |
| WA-06 | 夜间统计卡包含累计夜间用水、周期内最大峰值、周期内最小谷值；三张卡视觉样式一致且相互独立 |
| WA-07 | 峰谷时刻分布只统计全局夜间时段内的数据，横坐标随夜间时段配置变化 |
| WA-08 | 夜间用水不展示逐日明细和状态字段；相关导出数据也不包含状态字段 |

#### 夜间用水展示与交互要求

| 区域 | 展示内容 | 交互/口径 |
| --- | --- | --- |
| 周期筛选 | 昨日、本月、本年、自定义起止日期时间 | 默认本月；快捷切换立即刷新，自定义范围点击查询后刷新 |
| 累计夜间用水 | 当前查询周期内夜间用水累计值、统计周期数量、统一夜间时段 | 单位 m³ |
| 峰值流量 | 当前周期内最大峰值及发生日期、时刻 | 单位 m³/h |
| 谷值流量 | 当前周期内最小谷值及发生日期、时刻 | 单位 m³/h |
| 峰谷时刻分布 | 各夜间小时内峰值、谷值出现次数，以及最集中时段 | 横坐标按夜间时段配置生成，不展示全天无关时刻 |

#### 夜间用水验收标准

1. 首次进入夜间用水页签时，“本月”处于选中状态，统计数据与本月范围一致。
2. 切换昨日、本月、本年后，日期时间范围、三张统计卡和峰谷时刻分布同步变化。
3. 自定义日期时间选择器可正常切换年份、月份、日期和时间，查询后按新范围刷新。
4. 三张统计卡彼此独立，均采用与本页面其他信息面板一致的白底、细边框和统一圆角样式。
5. 配置夜间时段为 22:00-06:00 时，分布图横坐标只出现 22:00 至次日 06:00 的连续时刻。
6. 夜间用水页面不出现逐日明细入口，夜间明细及导出字段中均不出现“状态”。

### 4.3.3 数据报表

#### 主流程

1. 选择报表类型：日报 / 月报 / 季报 / 年报，及对应日期控件。
2. 选择片区（含子片区聚合）。
3. 点击「生成报表」→ 写入报表列表；同片区+类型+日期不可重复。
4. 导出 → 标记已导出（生产环境应对接文件下载）。

#### 用水量换算规则

| 报表类型 | 取数口径 |
| --- | --- |
| 日报 | dailyUsage |
| 月报 | monthlyUsage |
| 季报 | monthlyUsage × 3 |
| 年报 | monthlyUsage × 12 |

#### 报表导出功能

**功能定位**

报表导出用于将已生成的报表记录导出为文件，并标记导出状态，满足用水统计留档和外部流转需求。

**页面入口**

1. 入口位于"报表导出记录"列表每行操作列。
2. 只有存在报表记录时展示导出操作。
3. 导出动作基于该记录保存的报表类型、日期、片区、总用量和设备数。

**导出内容**

| 内容 | 说明 |
|------|------|
| 报表类型 | 日报、月报、季报或年报。 |
| 日期 | 当前报表统计周期。 |
| 片区 | 报表所属统计片区。 |
| 总用量(m³) | 当前周期总用水量。 |
| 设备数 | 纳入统计的大表数量。 |
| 单表明细 | 按设备列出用水数据。 |

**页面交互逻辑**

1. 用户点击"导出"后，系统执行报表导出。
2. 导出成功后提示"报表导出成功"。
3. 已导出的记录状态更新为"已导出"。
4. 重复点击导出应以同一报表记录为准，不重复生成统计记录。

**验收要点**

1. 导出操作只针对当前行记录。
2. 导出成功后状态正确变化。
3. 导出不会改变查询条件和片区选择。

**导出文件命名规范**

本规范适用于大表中心 - 数据报表模块的所有报表导出功能。

- **命名格式**：`{片区名称}_{报表类型}_{统计日期}.{文件格式}`

- **字段说明**：
  - **片区名称**：当前选中片区的名称；"全部片区"时使用 `全部片区`；"未分配区域"时使用 `未分配区域`
  - **报表类型**：日报 / 月报 / 季报 / 年报
  - **统计日期**：根据报表类型格式化
    - 日报：`YYYY-MM-DD`（如 `2026-07-13`）
    - 月报：`YYYY-MM`（如 `2026-07`）
    - 季报：`YYYY-QQ`（如 `2026-Q3`）
    - 年报：`YYYY`（如 `2026`）
  - **文件格式**：`.xlsx`、`.csv`

- **完整示例**：
  - `华东区_日报_2026-07-13.xlsx`
  - `华北区_月报_2026-07.xlsx`
  - `全部片区_季报_2026-Q3.xlsx`
  - `全部片区_年报_2026.xlsx`
  - `未分配区域_日报_2026-07-13.xlsx`

- **特殊规则**：
  1. 片区名称包含特殊字符时：保留原名称，不做转义或替换（如 `华东-A区_日报_2026-07-13.xlsx`）
  2. 同名文件冲突：浏览器下载时自动追加序号（如 `华东区_日报_2026-07-13 (1).xlsx`），由浏览器行为决定，前端不额外处理
  3. 文件格式：当前默认导出 `.xlsx` 格式；若后续支持 CSV，则扩展名相应改为 `.csv`

- **技术实现要点**：

在 `DataReportPage.tsx` 的 `handleExport` 函数中：

```typescript
const handleExport = (reportId: string) => {
    const report = reports.find(r => r.id === reportId);
    if (!report) return;

    // 构建文件名
    const areaName = getAreaNameForExport(report.areaId, areas); // 获取片区名称
    const dateStr = formatDateForExport(report.date, report.type); // 格式化日期
    const fileName = `${areaName}_${report.type}_${dateStr}.xlsx`;

    // 触发下载（实际项目中调用后端接口或生成 Blob）
    downloadFile(fileName, report.data);

    // 更新导出状态
    setReports(prev => prev.map(r =>
        r.id === reportId ? { ...r, exported: true } : r
    ));
    showToast('报表导出成功', 'success');
};
```

### 4.3.4 区域配置

#### 业务目标

维护大表业务的片区树，并把未绑定的大表业务设备关联到具体片区，形成后续数据监测、报表统计和档案追踪的空间归属。

#### 主流程

1. 左侧片区树 + 右侧详情 Tab（基本信息 / 子片区 / 关联设备）。
2. **片区 CRUD**：新增指定 parentId；编辑改名称；新增/编辑从右侧抽屉打开；删除前检查是否有子片区或关联设备。
3. **片区 ID**：新增时系统生成只读 ID，格式为 `AREA-YYYYMMDD-三位流水`，例如 `AREA-20260709-001`；编辑名称不改变片区 ID。
4. **设备绑定**：仅可选择未绑定且产品分类业务类型为「大表」的设备 → 写 `largeMeterAreaId` → 同步 largeMeters → 首次绑定并补全安装信息时生成「安装投运」档案。
5. 支持批量绑定、导入、解绑、编辑安装信息（用户/点位/厂家等）。
6. 关联设备列表「查看档案」→ 跳转大表档案并定位该设备。
7. **安装地址自动解析**：完善安装信息时，具体地址由经纬度通过百度地图逆地理编码自动解析得到，无需手动输入；地址与经纬度一同展示在「位置信息」区块中，取消独立的"重新选点"入口。

#### 业务规则

| 规则编号 | 规则描述 |
| --- | --- |
| AR-01 | 有子片区的节点不可删除 |
| AR-02 | 有关联设备的节点不可删除 |
| AR-03 | 安装信息变更触发档案：用户变更 → user-change；地址/坐标变更 → location-change |
| AR-04 | 表具/远传厂家选项来自系统字典 |
| AR-05 | 绑定设备列表只展示未绑定的大表业务设备；大表业务设备按产品分类业务类型判断 |
| AR-06 | 片区 ID 由系统生成且只读，不使用片区名称拼音或英文缩写，避免改名后 ID 失真 |
| AR-07 | 安装地址由经纬度自动解析，不开放手动编辑；无经纬度时地址为空，需先完成设备选点 |

### 4.3.5 大表档案

#### 业务目标

记录单台大表从设备接入、安装投运到后续维护的完整业务履历，区别于设备日志（通信/运行数据）。设备一接入平台即应生成基础档案，不应等到区域绑定后才建档。

#### 档案记录类型

| 类型 | 来源 | 说明 |
| --- | --- | --- |
| access | 系统自动 | 设备接入平台，生成基础设备档案 |
| installation | 系统自动 | 首次区域绑定并补全用户、安装时间、厂家等信息后，记录安装投运；安装地址由经纬度自动解析 |
| user-change | 系统自动 / 人工补录 | 安装信息中的用户号/名变化可自动留痕；也可人工新增用户变更记录 |
| location-change | 系统自动 / 人工补录 | 安装地址或坐标变化可自动留痕；也可通过地图选点人工补录位置变更 |
| status-change | 系统自动 | 启用/停用状态变更 |
| accessory | 人工补录 | 记录更换原因、原配件和新配件信息 |
| maintenance | 人工补录 | 记录维修保养项目、处理过程和现场结果 |
| calibration | 人工补录 | 记录检定项目、检定过程和独立的检定结果 |
| other | 人工补录 | 其他说明 |

#### 主流程

1. 档案总览：检索大表业务设备 → 进入单设备时间线；未分配片区的设备也应可见。
2. 时间线按发生时间倒序排列，最新记录在最上方；单台设备的业务发生顺序必须符合“设备接入 → 安装投运 → 后续变更、维护和检定”。
3. 按记录类型筛选；查看记录详情。用户、位置、配件和启停等变化展示变更前/变更后；设备接入、安装投运不展示变更前/变更后；检定校准单独展示“检定结果”。
4. 新增人工记录：必填发生时间和对应记录内容；可选附件（生产环境应对接文件存储）。用户变更记录填写变更原因、原用户名称和新用户名称，不要求输入用户编号。
5. 编辑人工记录：在当前时间线节点内展开编辑表单，不跳转到时间线顶部；记录类型只读，不允许切换为其他类型。
6. 列表中的「安装时间」口径调整为「接入时间」；时间格式统一为 `YYYY-MM-DD HH:mm:ss`，例如 `2026-07-08 00:37:00`。

#### 新增档案记录

##### 功能定位与适用范围

新增档案记录用于补录系统无法自动产生的设备履历。新增入口位于单设备档案抽屉的记录筛选区右侧，点击后在时间线上方展开表单。

- 可新增类型：配件更换、用户变更、位置变更、维修保养、检定校准、其他记录。
- 不可新增类型：设备接入、安装投运、启停变更。以上类型必须由对应设备操作自动生成，避免人工伪造系统生命周期事件。
- 新增成功后的记录来源固定为“人工补录”，操作人取当前登录用户名称。

##### 通用字段

| 字段 | 必填 | 编辑方式 | 规则 |
| --- | --- | --- | --- |
| 记录类型 | 是 | 下拉选择 | 只展示允许人工补录的 6 种类型；切换类型时清空上一类型的专属字段 |
| 发生时间 | 是 | 日期时间选择 | 默认当前时间，保存格式统一为 `YYYY-MM-DD HH:mm:ss`；用于时间线倒序排列 |
| 操作人 | 系统带入 | 只读 | 展示当前登录用户名称，不允许手工修改 |
| 附件 | 否 | 多文件选择 | 支持一次选择多个附件；保存后在当前记录中展示附件数量和查看入口 |
| 备注 | 否 | 多行文本 | 用于补充现场情况、资料说明、后续计划等非结构化信息 |

##### 类型专属字段与校验

| 记录类型 | 标题规则 | 必填字段 | 选填字段 | 展示结果 |
| --- | --- | --- | --- | --- |
| 配件更换 | 使用“更换配件”作为记录标题 | 更换配件、更换原因及处理过程、原配件信息、新配件信息 | 备注、附件 | 时间线展示原配件与新配件的变更前后对比 |
| 用户变更 | 标题固定为“用户变更” | 变更原因、新用户名称；原用户名称从设备当前安装/绑定信息自动获取 | 备注、附件 | 原用户名称只读；时间线展示原用户与新用户，不要求输入或拼接用户编号 |
| 位置变更 | 标题固定为“位置变更” | 变更原因、新安装位置、新经度、新纬度 | 备注、附件 | 通过地图选点带入新地址和坐标；原地址、原经度、原纬度从设备当前档案自动记录 |
| 维修保养 | 使用“维护项目”作为记录标题 | 维护项目、维护内容、处理结果 | 备注、附件 | 时间线展示维护过程和处理结果，不强制套用变更前后语义 |
| 检定校准 | 使用“检定项目”作为记录标题 | 检定项目、检定过程、检定结果 | 备注、附件 | 时间线独立展示“检定结果”，不使用“变更前/变更后”标签 |
| 其他记录 | 使用“记录标题”作为标题 | 记录标题、记录内容 | 备注、附件 | 按通用记录展示，不强制展示变更前后信息 |

##### 新增交互流程

1. 点击“新增记录”，初始化空表单，发生时间默认为当前时间，操作人自动带入当前登录用户。
2. 选择记录类型后，页面切换为该类型的专属字段、占位提示和必填规则；切换类型会清空上一类型已填写的专属内容，防止字段串用。
3. 仅已绑定用户的设备允许选择“用户变更”；原用户名称从设备当前安装/绑定信息自动获取并只读展示，字段下方不额外显示取值提示。设备未绑定用户时不展示“用户变更”类型，也不提供用户变更记录的编辑入口。
4. 选择“位置变更”时，点击“选择新位置”打开地图；确认点位后回填新安装位置、经度和纬度。经度必须在 -180～180，纬度必须在 -90～90。
5. 点击“选择附件”可多选文件；未选择附件不影响保存。
6. 点击“保存记录”后一次性校验全部必填项。缺失时提示具体字段名称并保留当前表单内容。
7. 校验通过后生成新记录，来源写为“人工补录”，关闭表单并刷新时间线；记录按发生时间重新排序，不按创建先后强行置顶。
8. 点击“取消”或“收起”放弃本次未保存内容，不新增档案记录。

##### 新增验收标准

1. 新增类型中不出现设备接入、安装投运和启停变更。
2. 6 种人工补录类型的字段名称、必填项和提示文案存在明确差异。
3. 用户变更、位置变更、配件更换保存后能正确展示变更前后信息。
4. 检定校准保存后能独立展示检定结果。
5. 位置变更不要求用户重复输入原位置和原坐标，且坐标范围校验有效。
6. 操作人不可修改，保存后的来源为“人工补录”。
7. 新增后时间线仍保持最新发生时间在最上方。

#### 编辑档案记录

##### 功能定位与适用范围

编辑档案记录用于修正人工补录记录中的时间、业务内容、附件或备注。是否可编辑以记录来源为准，不仅以记录类型判断。

- 来源为“人工补录”的配件更换、用户变更、位置变更、维修保养、检定校准和其他记录可编辑。
- 来源为“系统自动”的任何记录均不可编辑，包括设备接入、安装投运、启停变更，以及系统自动生成的用户或位置变更。
- 系统自动记录存在错误时，应修改设备或区域配置的源数据，由系统生成新的留痕记录，不直接篡改历史档案。

##### 编辑入口与展开位置

1. 可编辑记录在时间线卡片右上角展示“编辑”。
2. 点击“编辑”后，编辑表单在当前记录卡片内部、记录内容下方展开，不跳转到时间线顶部，也不打开独立弹窗。
3. 同一时间只编辑一条记录；打开另一条记录编辑时，切换到新的当前节点。
4. 取消或保存后关闭当前节点表单，并保持用户当前的时间线浏览位置。

##### 字段回显与可编辑性

| 字段 | 是否可编辑 | 编辑规则 |
| --- | --- | --- |
| 记录类型 | 否 | 回显原类型并置为不可更改状态；保存时强制沿用原类型，防止改变字段结构和业务语义 |
| 发生时间 | 是 | 修改后按新发生时间重新参与时间线倒序排列 |
| 类型专属业务字段 | 是 | 按原记录类型回显并沿用新增时的字段和必填校验规则 |
| 操作人 | 否 | 保留原记录操作人，不因当前编辑账号而改写 |
| 来源 | 否 | 始终保留“人工补录”，不可切换为“系统自动” |
| 附件 | 是 | 回显已有附件数量；可重新选择或补充附件，未执行附件调整时应保留原附件 |
| 备注 | 是 | 回显原备注，可修改或清空 |
| 记录 ID | 否 | 保存时沿用原记录 ID，不生成新记录 |

##### 编辑交互与保存规则

1. 点击编辑后回显原记录的发生时间、标题或固定标题、业务内容、变更前后信息、坐标、附件数量、操作人和备注。
2. 记录类型下拉处于禁用状态，并提示“编辑记录时不可更改记录类型”。
3. 编辑位置变更时，只允许重新选择新位置；原位置和原坐标作为历史事实保留，不开放手工修改。
4. 点击“保存修改”时，按原记录类型执行与新增一致的必填校验和坐标范围校验。
5. 保存成功后更新原记录，不新增副本；保留原记录 ID、来源、类型和操作人。
6. 若发生时间被修改，时间线立即按新时间重新排序；若筛选条件下记录不再匹配，应从当前筛选结果中移除。
7. 点击“取消”或“收起”放弃未保存修改，原记录保持不变。

##### 编辑验收标准

1. 系统自动记录不展示编辑入口，人工补录记录展示编辑入口。
2. 编辑表单准确出现在被点击的当前时间线节点内，不跳到顶部。
3. 编辑状态下记录类型不可选择，且异常操作也不能改变原类型。
4. 各类型专属字段完整回显，保存时执行正确的必填校验。
5. 保存修改不会改变原记录 ID、来源和操作人，也不会产生重复记录。
6. 修改发生时间后，时间线顺序立即更新并保持最新记录在最上方。
7. 取消编辑后原数据不发生变化，页面停留在当前记录附近。

#### 业务规则

| 规则编号 | 规则描述 |
| --- | --- |
| DA-01 | 大表档案设备范围与区域配置一致：只包含产品分类业务类型为「大表」的设备 |
| DA-02 | 设备接入即生成 access 档案记录，记录接入时间、设备身份、产品信息和基础位置 |
| DA-03 | 区域绑定时补全用户、表身号、安装时间、厂家等安装信息后，生成 installation 档案记录；安装地址由经纬度自动解析，无需手动填写 |
| DA-04 | 未绑定区域时，设备可能没有当前用户；当前用户信息以区域绑定/安装信息中填写的用户号和用户名为准 |
| DA-05 | 所有档案时间展示使用短横线日期格式并保留秒：`YYYY-MM-DD HH:mm:ss` |
| DA-06 | 「设备接入」记录在设备接入时自动生成，不可在大表档案中编辑或删除；人工补录记录（配件更换、维修保养、检定校准、其他记录）可编辑、可删除 |
| DA-07 | 系统自动生成的安装投运、用户变更、点位变更和启停变更记录只读；人工补录的用户变更、位置变更及其他记录可编辑、可删除 |
| DA-08 | 有附件的档案记录提供「查看附件」入口，展开后图片类附件弹窗预览（位于最上层），非图片附件提供「下载」文字按钮（无图标） |
| DA-09 | 人工新增用户变更时，“原用户名称”“新用户名称”均为必填项，只记录用户名称，不填写或拼接用户编号 |
| DA-10 | 单设备时间线按发生时间倒序排列，最新记录显示在最上方；接入时间必须早于安装投运时间，维护、配件、用户、位置、检定、启停和其他后续事件不得早于安装投运 |
| DA-11 | 设备接入和安装投运属于生命周期事件，不展示“变更前/变更后”；用户变更、位置变更、配件更换和启停变更按业务需要展示前后对比 |
| DA-12 | 检定校准必须记录并独立展示“检定结果”，不得仅将结果混入检定过程描述，也不得套用“变更前/变更后”标签 |
| DA-13 | 编辑人工补录记录时，编辑表单在当前记录节点内展开；记录类型不可修改，保存时沿用原记录类型、记录 ID、来源和原操作人 |
| DA-14 | 原型演示数据应覆盖设备接入、安装投运、配件更换、用户变更、位置变更、维修保养、检定校准、启停变更和其他记录全部 9 种类型，并保证各记录发生时间互不相同、业务顺序合理 |

#### 边界

- 不含：维护计划、审批流、批量导出、工单深度联动。
- 区域配置与设备管理是两个入口，读取同一份档案数据。

## 4.4 告警工单

告警工单模块实现「规则定义 → 告警产生 → 处置闭环 → 工单跟踪」链路。

### 4.4.1 设备告警信息

#### 业务目标

集中展示全平台设备告警实例，支持筛选、阅读、直接处理与转工单，形成告警处置入口；与综合视图、用水分析、消息推送联动。

#### 业务对象 — 告警

| 字段 | 说明 |
| --- | --- |
| eventName | 告警事件名称 |
| level | 紧急 / 重要 / 次要 / 提示 |
| deviceName / deviceCode | 触发设备 |
| productId | 所属产品 |
| space | 所属片区 |
| readStatus | 未读 / 已读 |
| processStatus | 未处理 / 处理中 / 已处理 |
| processMethod | 工单处理 / 直接处理 / 自动恢复 |
| triggerMethod | 设备状态触发 / 数据条件判断触发 / 事件上报时触发 / 功能调用时触发 / 属性数据上报时触发 |
| triggerConditions | 触发条件明细（多条件时可含逻辑：单条件 / 并集 / 交集） |
| content | 告警内容摘要（由触发条件自动生成） |
| triggeredAt | 触发时间 |
| workOrder | 转工单后嵌入的工单摘要（ID、名称、等级、处理人等） |

#### 状态流转

```text
未处理 ──直接处理（填写结果）──→ 已处理（processMethod=直接处理）
未处理 ──转工单────────────────→ 处理中（processMethod=工单处理，嵌入工单摘要）
处理中 ──查看关联工单──────────→ 处理中（等待工单办结）
已处理（自动恢复）────────────→ processResult 默认「自动恢复数据」
```

#### 主流程

**列表浏览**

1. 筛选区默认展示：告警等级、产品树、处理状态、关键词搜索；可**展开**更多筛选项（触发时间范围、触发方式、处理方式）。
2. 列表按触发时间倒序；分页展示。
3. 列表字段：序号、事件名称、告警等级、设备名称/编号、所属产品、所属片区、阅读状态、触发时间、触发内容、处理状态、操作。
4. 触发内容列展示截断文案，完整内容通过 title 悬浮或详情弹窗查看。
5. 支持从顶部通知铃铛、综合视图等入口带 `keyword` 或状态参数深链进入。

**单条操作**

1. **未处理**：处理 / 转为工单；点击后自动标记为已读。
2. **处理中 / 已处理**：查看；处理中且已转工单时可查看关联工单。
3. 处理 → 打开详情弹窗（process 模式），填写处理结果后提交。
4. 转为工单 → 打开转工单弹窗，填写工单名称、等级、处理人、描述 → 创建工单并置告警为「处理中」。

**批量操作**

1. 仅「未处理」告警可勾选；支持跨页保留选中（当前页全选）。
2. 批量处理：统一填写处理结果，批量置为「已处理 / 直接处理」。
3. 批量转为工单：统一配置等级、处理人、描述；若选中告警等级全部相同则继承该等级，否则默认「重要」；每条告警各生成一个工单。

#### 体验与交互要求

| 要求 | 说明 |
| --- | --- |
| 阅读状态 | 打开处理/查看/转工单时自动标记已读；未读以视觉标签区分 |
| 行选择 | 未处理行支持点击行选中；已处理/处理中行不可选 |
| 筛选折叠 | 常用筛选默认可见，高级筛选折叠以节省空间 |
| 详情弹窗 | 展示完整触发条件、设备信息、处理记录及关联工单 |
| 双入口 | 页面可从「告警工单」或「消息中心」侧边栏进入，面包屑与 Tab 上下文不同，列表能力一致 |

#### 业务规则

| 规则编号 | 规则描述 |
| --- | --- |
| ALM-01 | 批量转工单时，若选中告警等级全部相同则继承该等级，否则默认「重要」 |
| ALM-02 | 处理中状态强制 processMethod=工单处理 |
| ALM-03 | 大表类告警类型与数据监测告警类型枚举对齐 |
| ALM-04 | 仅未处理告警可勾选、可批量操作、可转工单 |
| ALM-05 | 转工单成功后告警嵌入 workOrder 摘要，可通过「查看工单」跳转工单管理 |
| ALM-06 | 关键词搜索匹配：事件名称、设备名称/编号、触发内容、告警内容、产品名称、所属片区 |
| ALM-07 | 告警触发设备编号与设备管理保持一致 |

#### 跨模块关系

- 综合视图告警处理率、待处理告警下钻 ↔ 本页 processStatus 统计。
- 用水分析「报警数据」Tab → 同设备告警列表，支持直接处理/转工单。
- 消息推送：`device-alarm` 类型推送源 + 对应模板 → 历史消息。
- 角色数据域 → 列表可见范围（生产环境按产品分类过滤，原型待实现）。

| RULE-02 | 开启「自动建工单」时必须配置指派人 |
| RULE-03 | 触发条件按触发方式校验字段完整性（如非整点上报需配置容差） |
| RULE-04 | 新建规则默认 enabled=true |

### 4.4.4 工单管理

#### 业务对象

| 字段 | 说明 |
| --- | --- |
| type | 告警工单 / 其他工单 |
| status | 待处理 / 待验收 / 退回 / 已结单 |
| alarmId | 告警转工单时的来源告警 |
| assignees | 处理人列表 |
| space | 所属片区；告警工单列表与详情统一使用该字段名称 |

#### 状态流转

```text
待处理 ──处理人提交结果──→ 待验收
退回   ──处理人重新提交──→ 待验收
待验收 ──管理员验收通过──→ 已结单
待验收 ──管理员验收不通过──→ 退回（需填写备注）
```

#### 主流程

1. 工单列表筛选：状态、类型、时间、关联设备/关键词。
2. **告警转工单**：从告警列表发起，自动填充设备与告警信息。
3. **新建其他工单**：手动选择关联设备、处理人、描述与优先级。
4. **处理**：处理人（指派人）提交处理结果与附件 → 进入待验收。
5. **验收**：管理员验收通过/不通过 → 已结单或退回。

#### 业务规则

| 规则编号 | 规则描述 |
| --- | --- |
| WO-01 | 处理权限：当前用户为指派人，且状态为待处理或退回 |
| WO-02 | 验收权限：管理员角色，且状态为待验收 |
| WO-03 | 可见性：管理员看全部；处理人只看 assignees 含自己的工单 |
| WO-04 | **待产品确认**：工单结单后是否自动将关联告警置为「已处理」 |
| WO-05 | 告警工单列表和详情中的空间归属字段统一命名为“所属片区”，不使用“所属区域” |

#### 跨模块关系

- 告警 → 工单：`createWorkOrderFromAlarm`
- 用水分析报警 Tab → 同上
- 综合视图告警处理率 ↔ 告警 processStatus 统计
- 消息推送：`device-alarm` 类型推送源 + 对应模板

## 4.5 消息中心

消息中心负责「配置推送策略 → 定义消息模板 → 记录发送历史」。

### 4.5.1 消息订阅

#### 业务对象 — 五类订阅

| 订阅 ID | 名称 | 推送内容 |
| --- | --- | --- |
| device-runtime | 设备运行情况分析 | 总数/在线/离线 |
| device-alarm | 设备告警消息分析 | 告警总数、分级、处理统计 |
| system-runtime | 系统运行情况分析 | CPU/内存/磁盘 |
| system-alarm | 系统告警消息分析 | 同设备告警结构 |
| work-order | 工单处理情况分析 | 工单总数、分级、处理统计 |

#### 主流程

1. 列表展示五类订阅，开关控制 enabled。
2. 配置全局推送时间（默认 06:00:00）与推送周期（每周一 / 每月 1 日等）。

---

### 4.5.2 推送源配置

#### 业务对象

- messageType：device-alarm / system-alarm / msg-subscribe / work-order
- platform：钉钉 / 企业微信 / 短信 / 邮件
- configItems：平台参数键值（如短信模板 ID）
- recipientUserIds：接收人

#### 主流程

1. 列表 CRUD 推送源。
2. 维护平台配置项（独立抽屉）与接收人（独立抽屉）。
3. 支持连通性测试与启用/禁用。

---

### 4.5.3 消息模板

#### 业务规则

| 规则编号 | 规则描述 |
| --- | --- |
| TPL-01 | 模板内容支持变量占位符 `#{key}` |
| TPL-02 | 内容长度不超过 maxLength（默认 200 字） |
| TPL-03 | 五类模板：device-alarm、system-alarm、device-runtime、alarm-report、work-order-report |

### 4.5.4 历史消息

#### 主流程

1. 筛选：操作类型、操作人、时间、操作结果、状态。
2. 查看详情：推送内容、目标用户、分用户投递结果（成功/失败）。

#### 推送链路


## 4.6 系统管理

系统管理模块支撑多租户平台治理，涵盖租户授权、组织人员、角色权限、系统配置与审计日志。

### 4.6.1 租户管理

#### 业务目标

维护租户层级结构，分配功能权限范围，支撑 SaaS 化多租户运营。

#### 主流程

1. 树形表格展示租户（根租户 + 子租户），支持展开/折叠。
2. 筛选：租户名称关键词搜索。
3. 新增子租户：指定上级租户，填写基本信息；继承父级管理员联系方式。
4. 编辑租户信息；**授权**抽屉勾选 `authorizedPermissionIds`（功能模块权限树）。
5. 查看租户详情；删除时检查是否有子租户。

#### 业务规则

| 规则编号 | 规则描述 |
| --- | --- |
| TEN-01 | 子租户权限不可超出父租户授权范围 |
| TEN-02 | 有子租户的节点不可删除 |
| TEN-03 | 新建子租户默认继承部分父级权限子集 |

---

### 4.6.2 用户管理

#### 业务目标

管理平台登录账号，关联租户、角色、部门与岗位，控制账号生命周期。

#### 主流程

1. 筛选：租户（树形选择）、角色、状态（正常/冻结）、关键词（账号/姓名）。
2. 选择租户后，角色下拉仅展示该租户下角色。
3. 列表展示：账号、姓名、租户、角色、部门、岗位、状态、最近登录等。
4. 新增/编辑：抽屉表单填写账号、姓名、租户、角色、部门、岗位；**角色必须属于所选租户**。
5. 单条操作：查看详情、编辑、重置密码、冻结/解冻、删除。
6. 批量操作：批量冻结、批量删除（superadmin 除外）。

#### 业务规则

| 规则编号 | 规则描述 |
| --- | --- |
| USR-01 | 账号全局唯一 |
| USR-02 | superadmin 不可冻结、不可删除 |
| USR-03 | 新用户默认密码为系统初始密码（参数管理配置，原型默认 `12345678`） |
| USR-04 | 切换租户时清空已选角色，防止跨租户角色错配 |
| USR-05 | 冻结账号拒绝登录 |

---

### 4.6.3 角色管理

#### 业务目标

定义租户内角色及其**功能权限**与**数据域**，驱动菜单可见性与业务数据范围。

#### 业务对象

| 对象 | 说明 |
| --- | --- |
| 功能权限 | 按模块 + 操作粒度（如 产品管理-新增、设备告警信息-导出），来源于租户权限树 |
| 数据域 | `alarmRuleCategoryScopes`：按产品分类树勾选可见范围；未配置或企业管理员视为全部 |
| 角色说明 | remark 字段，列表展示角色职责描述 |

**数据域生效范围**：设备列表、告警规则、工单管理、设备告警。

#### 主流程

1. 列表筛选：角色名称关键词；展示所属租户、角色编号、数据域摘要、角色说明、创建时间。
2. 新建/编辑角色 → 抽屉表单，含两个授权 Tab：
   - **菜单权限**：勾选功能权限树（模块 × 操作）。
   - **数据域**：勾选产品分类树叶子节点；不选视为全部数据域。
3. 企业管理员（isTenantAdmin）拥有全部功能权限与全部数据域，不可编辑权限配置。
4. 删除角色需二次确认。

#### 预置角色（示例）

| 角色 | 典型权限 |
| --- | --- |
| 企业管理员 | 租户内全部功能 + 全部数据域 |
| 大表管理员 | 综合视图、设备查看、大表中心、告警查看与规则维护；数据域=大表 |
| 户表管理员 | 综合视图、设备查看、告警查看与规则维护；数据域=户表 |
| 物联网角色 | 设备接入编辑、分组、地图、告警查看 |
| 运维人员 | 综合视图、设备查看、告警/工单全流程；规则只读 |
| 告警规则专员 | 告警规则与等级全权限 |
| 审计员 | 登录日志、操作日志只读 |

#### 业务规则

| 规则编号 | 规则描述 |
| --- | --- |
| ROL-01 | 企业管理员（isTenantAdmin）拥有全部功能权限与全部数据域 |
| ROL-02 | 非管理员角色可勾选功能权限树 + 数据域范围 |
| ROL-03 | 数据域影响：设备列表、告警规则、工单、设备告警的可见范围（列表页过滤待后端实现） |
| ROL-04 | 角色必须归属某一租户；新建时系统生成角色编号 |
| ROL-05 | 数据域未勾选任何分类时，视为「全部数据域」 |

## 5. 全局交互与布局

### 5.1 导航结构

- 顶部 Tab 切换五大模块；切换时进入各模块默认页。
- 左侧 Sidebar 展示当前模块功能菜单；支持分组折叠。
- 面包屑展示当前页面路径。
- Hash 路由格式：`#page=<pageId>`，支持带参数深链（如 productId、groupId、meterId）。

### 5.2 通知与用户

- 顶部铃铛：告警/公告通知下拉；点击可带 keyword/status 筛选跳转对应模块。
- 用户区域：展示当前用户、退出登录。

### 5.3 权限与菜单

- 顶栏 Tab 按用户 permissionIds 控制可见性。
- 侧边栏菜单与页面内操作按角色功能权限控制（原型部分页面尚未完整过滤）。

### 5.4 通用组件模式

- 列表页：搜索 + 筛选 + 表格/卡片 + 分页。
- 表单页：分步或 Tab 表单，保存/取消/返回。
- 详情：抽屉或独立页面，支持编辑模式切换。
- 操作反馈：Toast 提示、二次确认对话框。

## 9. 验收标准

1. **登录**：验证码 + 账号密码校验；冻结账号拦截；记住账号生效。
2. **设备接入**：产品/设备 CRUD 及校验规则生效；产品分类树与列表联动；卡片/列表视图切换；分组 tags 同步；地图定位与回写；OTA 任务状态流转；接入配置引用检查（产品删协议）。
3. **综合视图**：全局筛选生效；快照/趋势指标口径正确；健康指标阈值与状态标签正确；概况下钻抽屉与 CSV 导出可用；不可下钻项无误导链接；设备接入流程快捷入口可跳转。
4. **大表中心**：状态展示规则（告警显示具体类型）；夜间用水配置生效；报表去重；区域绑定触发档案与 largeMeters 同步；档案自动/人工类型完整。
5. **告警工单**：告警阅读状态与状态机（未处理→处理中/已处理）正确；筛选展开/收起；批量处理与批量转工单；转工单联动；工单四状态流转与权限分离；等级 CRUD 引用约束。
6. **消息中心**：五类订阅开关；推送源与模板配置；历史消息投递详情。
7. **系统管理**：租户树与授权；用户-租户-角色一致性及批量操作；角色功能权限与数据域双 Tab 配置；部门树与用户关联；预置角色权限符合预期。
8. **标注文档**：标注面板可阅读本 PRD 各章节，页面导航可跳转。
