# 设备综合视图重新规划

## 1. 已确认定位

- 目标用户：设备运维管理层。
- 页面性质：普通电脑端管理工作台，并非展示型大屏。
- 核心任务：快速判断资产规模、运行健康、数据质量风险和告警工单闭环情况。
- 设计方向：延续现有平台，以管理结论和风险下钻为主，不做图表拼盘。

## 2. 当前页面的主要问题

现有页面将设备概览、告警工单、服务器状态、数据流入流出和排行并列展示，但缺少统一的管理主线：

1. 资产、连接、数据和处置指标彼此割裂，无法快速形成整体判断。
2. 数据流量与服务器状态偏技术监控，不能直接说明业务风险。
3. 只有结果数字，缺少同比/环比、趋势、异常原因和影响范围。
4. 异常指标没有形成从统计到设备、告警、工单的连续下钻路径。
5. “在用/闲置”与“在线/离线”是不同维度，现有概览容易混淆资产使用状态和连接状态。

## 3. 新版信息主线

```text
资产家底 → 运行健康 → 数据质量 → 告警与工单闭环
```

首屏优先回答四个管理问题：

1. 当前有多少设备资产，结构是否合理？
2. 设备整体运行是否健康，问题是否正在扩大？
3. 哪些数据异常正在影响管理和业务判断？
4. 告警是否得到及时处置，工单是否形成闭环？

## 4. 页面结构

### 4.1 顶部范围与口径

- 页面标题：设备综合视图。
- 全局筛选：统计时间、产品。
- 辅助信息：数据更新时间、自动刷新状态、指标口径说明。
- 默认时间：近 7 天；资产存量显示当前快照，趋势类指标跟随时间筛选。

筛选条件应作用于全页；进入明细后保留当前筛选上下文。

### 4.2 管理摘要

采用一组紧凑的核心指标，不使用夸张大数字：

| 指标 | 管理含义 | 建议辅助信息 |
|---|---|---|
| 设备总量 | 当前资产规模 | 较上期新增/减少 |
| 在用设备 | 已投入业务使用的资产 | 在用率 |
| 闲置设备 | 已建档但未投入使用的资产 | 闲置超过 30 天数量 |
| 在线率 | 当前连接健康程度 | 较上期变化、离线数 |
| 数据正常率 | 按要求正常上报的设备占比 | 异常设备数 |
| 告警处理率 | 已处理告警占告警总量的比例 | 待处理告警数 |

同时给出一句“管理摘要”，例如：

> 当前设备在线率 96.8%，较上期下降 1.2 个百分点；主要受 18 台长期离线设备影响，其中 7 台尚未生成工单。

摘要由规则生成，只描述最重要的 1 至 2 个变化或风险。

### 4.3 设备资产概况

目标是看清资产结构，不只是总数。

- 资产状态：在用、闲置及占比。
- 产品分布：各产品设备数量与占比。
- 分组分布：各分组设备数量与占比。
- 可选补充：新增设备趋势、长期闲置设备清单。

交互：切换“按产品/按分组”；点击任一分类进入设备管理列表。

### 4.4 在线/离线分析

目标是判断连接健康和风险走势。

- 核心指标：在线设备数、离线设备数、在线率、长期离线数。
- 趋势：按小时/日展示在线率变化，可切换产品或分组维度。
- 离线时长分布：小于 1 小时、1 至 24 小时、1 至 3 天、超过 3 天。
- 风险列表：长期离线设备 Top 5，展示设备、产品/分组、离线时长和当前工单状态。

交互：趋势点和时长区间均可下钻到设备列表；长期离线设备可跳转设备详情或关联工单。

### 4.5 数据接收异常

目标是区分数据质量问题的来源和影响。

- 核心指标：异常设备数、异常上报次数、数据正常率、较上期变化。
- 异常分类：解析失败、缺报、非整点上报。
- 趋势：各异常类型随时间的变化。
- 分布：按产品、分组统计异常设备占比。
- 风险列表：连续异常设备 Top 5，展示最近异常、连续时长和影响程度。

当前系统没有独立的数据异常明细页，因此“数据正常率”只定位到本页的数据接收异常模块；解析失败、缺报、非整点上报暂不配置跨页跳转，待明确明细数据与承载页面后再接入。

建议口径：同一设备在一个统计周期内出现多次异常时，“异常设备数”去重，“异常次数”累计，两者不可混用。

### 4.6 告警与工单闭环

目标是判断风险是否及时处理，而不是只看告警数量。

- 告警总量、未处理告警、已转工单告警、未转工单告警。
- 工单总量、处理中、已办结、超时未办结。
- 处置漏斗：告警产生 → 已确认 → 已转工单 → 已办结。
- 效率指标：告警确认及时率、工单办结率、平均办结时长。
- 高频风险：高频告警设备类型、重复告警设备 Top 5。

交互：点击漏斗任一阶段进入告警或工单列表；高频设备类型可继续下钻到具体设备。

## 5. 页面布局建议

面向 1440px 左右的普通办公屏幕，采用纵向工作台：

1. 顶部：标题、全局筛选、更新时间。
2. 第一屏：设备运行概况。
3. 第二段：资产概况 + 在线/离线分析。
4. 第三段：数据接收异常 + 告警与工单闭环。

页面允许纵向滚动。核心结论与待关注事项应在首屏可见；明细表通过跳转或侧边详情承载，不把整张设备表塞进综合视图。

## 6. 关键状态与规则

- 正常：展示最新数据和更新时间。
- 加载：各模块独立骨架加载，避免全页阻塞。
- 空数据：说明当前筛选范围无数据，并提供清除筛选入口。
- 数据延迟：明确标记最后更新时间和延迟范围。
- 指标异常：状态同时使用颜色、图标和文字，不只依赖颜色。
- 权限不足：隐藏无权限的跳转操作，但保留可查看的汇总结果。

## 7. 数据来源

优先复用现有平台数据：

- 设备、产品、分组：设备接入模块。
- 在线状态和离线时长：设备连接状态及上下线记录。
- 解析失败、缺报、非整点上报：设备运行数据接收与校验记录。
- 告警：设备告警模块。
- 工单：工单管理模块及告警关联关系。

需要后端进一步确认的口径：在用/闲置判定、长期离线阈值、整点上报容差、告警闭环定义、及时率时限和超时规则。

## 8. 本轮边界

本轮包含：产品结构、模块内容、指标建议、关键交互、状态和数据来源规划。

本轮不包含：可视化样式定稿、页面代码、后端统计接口、预警算法和自动派单策略。

## 9. 验收重点

1. 管理层进入页面后，10 秒内能判断整体规模、健康度、主要风险和处置情况。
2. 已有业务承载页的指标可下钻到对应设备、告警或工单；未建设明细页的数据异常指标不配置误导性跳转。
3. 在用/闲置与在线/离线口径明确且互不混淆。
4. 所有趋势和变化均标注时间范围及对比口径。
5. 全局筛选对全部模块生效，下钻后保留筛选上下文。
6. 页面在普通办公屏幕上可读、可操作，不采用大屏化视觉。

## 10. 已确认设计决策

- 第一轮曾选择 `src/themes/incident/DESIGN.md`，实际预览后确认其黑色与橙红表达和现有系统不协调。
- 最终视觉基底调整为现有物联网平台自身的浅色、蓝色主色、细边框和紧凑后台密度。
- 删除“管理关注事项”，综合视图只保留资产、在线状态、数据质量、告警工单四类核心分析。
- “管理摘要”调整为“设备运行概况”：设备资产作为主信息块，在线率、数据正常率、告警处理率只展示核心数值、同期变化、状态和辅助说明；不使用进度条，也不配置跳转箭头和下钻操作。
- 设备资产概况使用环形占比图呈现在用/闲置比例；在线/离线分析使用环形占比图呈现连接状态，分类明细保留精确数量和占比。
- 状态判断统一使用“正常、关注、异常、无数据”。设备在线率阈值为 98%/95%，数据正常率阈值为 99%/97%，告警处理率阈值为 95%/85%；状态标签悬浮或键盘聚焦时展示阈值图和当前值位置。
