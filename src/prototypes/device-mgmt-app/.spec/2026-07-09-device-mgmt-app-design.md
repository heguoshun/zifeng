## 设计决策

### 1. 首页布局：待办数字卡片 + 快捷入口

- 顶部横排 **3 张待办卡片**：待处理告警 / 待办工单 / 异常设备（带数字）
- 卡片可点击，跳转对应 Tab 并带上筛选状态
- 下方 **3 个快捷入口**，分别进入设备 / 告警 / 工单列表
- 底部固定 **5 Tab** 导航

### 2. 列表样式：统一卡片列表

- 设备、告警、工单三个列表页均采用卡片列表
- 卡片结构：左侧状态色条或类型图标 → 中间名称+关键信息 → 右侧时间/状态标签
- 顶部固定搜索栏 + 筛选标签（状态、设备类型、区域等）

### 3. 详情页：整页滚动分区

- 顶部：状态标签 + 关键摘要（设备名、告警等级、工单状态、时间）
- 中间：分区展示详细信息（基本属性、实时读数、关联告警/工单）
- 操作按钮放在页面内容流中（告警：确认/关闭/转工单；工单：接单/完结）
- 不做底部固定操作栏（与推荐方案略有不同，按用户选择）

### 4. 状态色：对齐 PC 平台语义

| 语义 | 颜色 | 用途 |
|------|------|------|
| 主色/品牌 | `#1890ff` 蓝 | 主按钮、选中态、链接 |
| 紧急/离线 | 红 | 紧急告警、设备离线、超时 |
| 警告/待处理 | 橙 | 一般告警、待接单 |
| 正常/完成 | 绿 | 在线、已处理、工单完结 |

### 5. 预览形式：手机框预览

- 固定 iPhone 标准宽度 **390px**
- 外层加手机边框，评审时直观识别为移动端 App
- 底部 Tab 固定在手机框底部

### 6. 导航结构（继承需求决策）

底部 5 Tab：首页 / 设备 / 告警 / 工单 / 我的

## 页面清单（首版）

| 页面 | 说明 |
|------|------|
| 登录页 | 账号登录，沿用 PC 平台账号体系（原型模拟） |
| 首页 | 待办卡片 + 快捷入口 |
| 设备列表 | 卡片列表 + 搜索筛选 |
| 设备详情 | 水表/压力计分区信息 + 实时读数 |
| 告警列表 | 卡片列表 + 状态筛选 |
| 告警详情 | 详情分区 + 确认/关闭/转工单 |
| 工单列表 | 我的待办 + 状态筛选 |
| 工单详情 | 详情分区 + 接单/处理/完结 |
| 我的 | 消息通知列表 + 账号信息 |

## 数据与素材

- 模拟数据复用/参考 `iot-platform-home` 的 `devices.ts`、`deviceAlarms.ts`、工单相关数据
- 水表、压力计等设备类型与 PC 端产品分类保持一致

## 设计假设

- 操作按钮在详情页内容流末尾，用户需滚动到底部执行处置（非固定底栏）
- 首版不做地图、扫码、拍照上传
- 消息通知在「我的」Tab 内以列表呈现，Tab 角标显示未读数

## 实施边界（本轮）

**做：** 上述 9 个页面的可运行移动端原型，手机框预览，模拟数据交互

**不做：** 真实接口对接、推送通知、登录鉴权后端、地图/扫码、拍照、派单转派

## 建议验收

1. 手机框内可完整走通：登录 → 首页 → 三个模块列表 → 详情 → 处置操作
2. 首页 3 张待办卡片数字与列表筛选结果一致
3. 水表/压力计详情展示对应字段和读数
4. 告警可确认/关闭/转工单，工单可接单/完结
5. 视觉主色与 PC 平台一致，状态色语义统一
